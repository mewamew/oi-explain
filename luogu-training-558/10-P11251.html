[GESP202409 八级] 美丽路径

面向 GESP 八级练习的浅显题解。题面原文以洛谷为准，这里重点讲如何读题、如何一步步想到做法，以及代码怎样落地。

树直径BFS

0. 类型标签

树, 直径, BFS

1. 题意

树上一条路径相邻节点颜色必须不同，求最长这样的路径包含多少个节点。

2. 一点点思路提示

颜色相同的边不能走，把这些边删掉以后，问题就变成森林里的最长路径。

3. 解题思路

先读题抓信号路径要求相邻颜色不同，等价于同色边不能走。把不能走的边删掉后，剩下的就是若干棵树里的最长路径。

为什么不能暴力枚举所有点对再检查路径颜色，n 到 10^5 会超时。我们要把条件转成更简单的图结构。

先用几步抓住主线

  1. 只保留连接不同颜色节点的边，相同颜色的边直接忽略。
  2. 剩下的是若干棵小树，也就是一个森林。
  3. 每个连通块里的最长路径可以用两次 BFS 求直径：先从任意点找最远点，再从这个最远点找最远距离。
  4. 所有连通块直径的最大值加 1，就是最长美丽路径的节点数。

一步一步推到正解

一条路径美丽，当且仅当路径上的每条边两端颜色不同。也就是说，同色边永远不能出现在答案路径里。

把所有同色边删掉，原来的树会变成一个森林。森林里的每个连通块，任意两点之间的路径都由异色边组成，所以都是合法候选。

于是问题变成：在这些连通块中，找节点数最多的一条路径。对树来说，这就是直径。

求一棵树直径的简单方法是两次 BFS：从任意点出发找到最远点 a，再从 a 出发找到最远距离。

为什么两次 BFS 可以求直径？第一次找到的 a 一定位于某条最长路径的一端；第二次从 a 出发走到最远处，就能得到这条连通块里的最长距离。

注意题目里的长度是节点数量，而 BFS 得到的距离通常是边数，所以最后要加 1。

如果某个连通块只有一个点，它的直径边数是 0，答案节点数是 1。这也覆盖了全同色的情况。

代码怎么对应这个想法

  • 读边时只把颜色不同的边加入新图。
  • `seen` 标记每个连通块是否已经处理。
  • 第一次 BFS 同时标记连通块，第二次 BFS 求这个连通块直径边数。
  • 全局答案维护 `diameterEdges + 1` 的最大值。

复杂度：每条保留下来的边参与 BFS 常数次，O(n)。

4. 用测试样例验证解题思路

样例一删掉同色边后，路径 2-1-3-4 仍然连通，边数为 3，所以节点数输出 4。

样例 1 输入

5
1 0 0 1 0
1 2
3 5
4 3
1 3

样例 1 输出

4

样例 2 输入

5
0 0 0 0 0
1 2
2 3
3 4
4 5

样例 2 输出

1

容易错的地方

  • 题目里的长度是节点数，不是边数。
  • 如果所有边都被删掉，单个节点本身也是长度 1 的路径。
