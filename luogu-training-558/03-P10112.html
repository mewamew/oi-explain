[GESP202312 八级] 奖品分配

面向 GESP 八级练习的浅显题解。题面原文以洛谷为准，这里重点讲如何读题、如何一步步想到做法，以及代码怎样落地。

组合计数排列阶乘取模

0. 类型标签

组合计数, 排列, 阶乘, 取模

1. 题意

把若干种奖品分给 N 个有编号的同学，每人一个，奖品可以刚好用完，也可能剩 1 个，求分配方案数。

2. 一点点思路提示

同学是有编号的，所以不是只看每种奖品用了几个，还要看这些奖品分别给了哪些同学。

3. 解题思路

先读题抓信号题面说每位同学都有学号，奖品同种之间没有区别，要求“方案数取模”。这说明它不是普通模拟，而是组合计数题。

为什么不能暴力如果逐个同学尝试分奖品，N 到 1000，分支数量会爆炸。我们必须直接数方案，而不是真的枚举每种分法。

先用几步抓住主线

  1. 如果所有奖品刚好 N 个，就是经典的“重复物品排列”：先把 N 个位置排好，再除掉同种奖品内部交换造成的重复。
  2. 如果奖品总数是 N+1，就一定会多出 1 个。可以枚举哪一种奖品多出来，等价于这一种少用 1 个。
  3. 把枚举式整理一下，会发现只需在刚才的基础上乘上奖品总数，也就是 N+1。
  4. 因为要取模，除法不能直接做，要预处理阶乘和逆元阶乘。

一步一步推到正解

先想最简单的情况：奖品总数正好等于 N。N 个有编号的同学可以看成 N 个位置，我们要把多种奖品填进这些位置。

如果所有奖品都不同，排列数是 N!。但同一种奖品之间没有区别，交换两个同种奖品不会产生新方案，所以要把每种奖品内部的重复排列除掉。

于是得到 `N! / (a0! a1! ... )`。这里不需要孩子先懂高深公式，只要理解：先把所有位置排满，再把“同种奖品互换造成的重复”去掉。

再看奖品总数是 N+1 的情况。最后会剩下一件奖品。可以想成：从 N+1 件奖品里挑一件不发，剩下 N 件按刚才的方法分。

这里有一个很容易卡住的点：最后乘的不是奖品种类数 M，而是奖品总数。原因是某一种奖品如果有 a 个，那么“少发一个这种奖品”会让它贡献 a 份，而不是只贡献 1 份。

把每一种奖品的贡献加起来，就是 a0+a1+...，也就是代码里的 `sum`。所以当 `sum == N + 1` 时，代码用 `ans = ans * sum % MOD` 把“多出来 1 个奖品”的所有情况一次性补上。

比如 N=3，奖品数量是 1 和 3。基础值先算出 1；如果剩下第 1 种奖品有 1 种，如果剩下第 2 种奖品有 3 种，合起来就是 4 种，也就是乘上总数 4。

因为答案要对 10^9+7 取模，普通除法不能直接写。代码里把除以 a! 写成乘以 a! 的逆元。

代码怎么对应这个想法

  • `fact[i]` 保存 i!，`invFact[i]` 保存 i! 在模意义下的倒数。
  • 每组先算 `fact[N]`，再乘每个 `invFact[a_i]`。
  • `sum` 记录所有奖品一共有多少个。如果奖品总数是 N+1，再乘上 `sum`，表示枚举哪一个奖品不发。
  • `if (sum == N + 1) ans = ans * sum % MOD;` 这一行就是专门处理“多 1 个奖品”的情况；后面的 `% MOD` 是为了按题目要求取模。
  • 模数是质数，所以逆元可以用快速幂 `pow(x, MOD-2)` 求。

复杂度：预处理 O(MAX)，每组数据 O(M)。

容易错的地方

  • 总奖品数只可能是 N 或 N+1，这个条件很关键。
  • 取模下的除法要写成乘逆元。
