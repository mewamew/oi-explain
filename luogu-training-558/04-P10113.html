[GESP202312 八级] 大量的工作沟通

面向 GESP 八级练习的浅显题解。题面原文以洛谷为准，这里重点讲如何读题、如何一步步想到做法，以及代码怎样落地。

树LCA倍增路径最大值

0. 类型标签

树, LCA, 倍增, 路径最大值

1. 题意

公司上下级形成一棵树，每次给出一组员工，找能管理所有人的最靠下的共同领导。

2. 一点点思路提示

能管理一群人的人一定在他们共同祖先那条线上。先找到最靠下的共同祖先，再在它到老板的这条线上找编号最大的员工。

3. 解题思路

先读题抓信号员工和直接领导构成一棵以 0 为根的树。题目问一组点的共同管理者，本质是找共同祖先；又要求编号最大，所以还要注意祖先链上的最大编号。

为什么不能暴力如果每次把每个参与者的所有祖先都列出来再求交集，m 到 10^4、N 到 10^5，虽然 Q 不算特别大，但仍然容易写慢，也不够稳。

先用几步抓住主线

  1. 把老板 0 看成树根，直接领导就是父节点。
  2. 先预处理每个节点往上跳 1、2、4、8 步会到哪里，这就是倍增表。
  3. 求一组员工时，先拿第一个人当当前答案，再依次和下一个人求 LCA。
  4. LCA 合并完以后，得到能覆盖整组员工的最靠下公共管理范围；所有能主持的人就是这个点和它往上的祖先。
  5. 题目要编号最大的主持人，所以最后输出从老板到这个 LCA 路径上的最大编号。

一步一步推到正解

先理解管理关系：x 能管理 y，等价于 x 是 y 自己或者 y 的祖先。要能管理一组人，就必须是这组人的共同祖先。

两个人最靠下的共同祖先就是 LCA。一群人也一样，可以先求前两个人的 LCA，再拿这个结果和第三个人求 LCA，一直合并下去。

合并得到的节点代表“最靠下的公共管理范围”。这个节点往上的所有祖先也都能管理整组人。

题面要的是编号最大的主持人，而不是最靠下的主持人。很多树题里父亲编号天然较小，但这里更稳的理解是：在老板到这个 LCA 的路径上，找编号最大的节点。

所以预处理时除了 LCA 倍增表，还记录 `pathMax[x]`，表示从根 0 到 x 这条链上的最大编号。

每次询问得到共同 LCA 后，输出 `pathMax[lca]`。这样不依赖隐藏的编号规律，更贴合题意。

代码怎么对应这个想法

  • `parent[i]` 是员工 i 的直接领导，建出孩子列表后从 0 开始遍历。
  • `up[j][x]` 支持把 x 往上跳 2^j 层，用来求 LCA。
  • `pathMax[v] = max(pathMax[parent], v)` 在建深度时顺便算出。
  • 询问里把所有参与者依次合并成一个 LCA，最后输出 `pathMax[ans]`。

复杂度：预处理 O(N log N)，每个参与员工合并一次 O(log N)。

4. 用测试样例验证解题思路

样例中员工 3 和 4 的公共管理者有 2 和 0，编号最大的就是 2；员工 1 和 4 只能一起归到 0，所以输出 0。

样例 1 输入

0 0 2 2
3

3 2 3 4
2 1 4

样例 1 输出

2
0

5

样例 2 输出

1
1
1

容易错的地方

  • 节点从 0 开始编号，老板是 0。
  • 一组里 m 可以很大，要让每次合并只花 logN。
  • 如果只输出 LCA，会默认领导编号都比下属小；按路径最大值写更贴合题面。
