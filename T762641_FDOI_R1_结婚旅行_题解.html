「FDOI-R1」结婚旅行

树上问题 DFS 序 树状数组 线段树 位运算 动态插入 可达性分类
01
Background

题目背景

小 P 和小 L 刚刚结婚，为了庆祝新婚，他们决定在 X 国进行一次难忘的结婚旅行。

02
Problem Statement

题目描述

X 国有 $n$ 个城市，编号 $1$ 至 $n$，由 $n-1$ 条双向航线连接，城市之间可以通过航线直接或间接抵达。

小 P 和小 L 从城市 $1$ 出发。他们事先规划了 $m$ 条旅游路线，这些路线被分为 $C$ 种颜色。第 $i$ 条路线是起点为 $1$ 终点为 $k_i$ 的唯一简单路径，该路线的颜色为 $c_i$（$1 \le c_i \le C$）。

他们将进行若干次旅行。每次旅行会给出 $p_i$ 条路线（可能重复选择同一条路线）。对于一次旅行，他们希望只访问 $t_i$ 种颜色的路线，而其他颜色的路线在本次旅行中累计出现次数不超过 $z_i$。

为了达成这个目标，对于每条被选中的路线 $S$，当且仅当这条路线 $S$ 的终点是另外一条路线 $T$ 的终点在树上的祖先或孙子（即 $S$ 为 $T$ 的前缀或 $T$ 为 $S$ 的前缀），$S$ 可以被替换为 $T$。每次旅行的每条路线至多只能进行 $a_i$ 次这样的操作。

现在小 P 与小 L 希望你能告诉他们，他们每一次旅行能否达到他们的要求。

§
形式化题意

给你一颗 $n$ 个点的树，共有 $C$ 种颜色，初始没有任何路线。先给出 $m$ 条路线，每条路线是 $1$ 为起点，$k_i$ 为终点的简单路径，颜色为 $c_i$。随后给出 $q$ 次操作，共有两种操作类型：

  1. 修改（$d_i = 1$）：添加一条起点为 $1$，终点为 $k_i$，颜色为 $c_i$ 的路线，新路线编号为 $cnt+1$，然后 $cnt$ 自增 $1$。
  2. 查询（$d_i = 0$）：给你 $p_i$ 个路线与 $t_i$ 种想要的颜色。每个路线可进行不超过 $a_i$ 次操作（将当前路线转化为另一有颜色的路线，满足前缀关系）。求转化后能否满足至多有 $z_i$ 个路线不存在任意一种想要的颜色。
03
Input Format

输入格式

第一行三个整数 $n, m, C$。

接下来 $n-1$ 行，每行两个整数 $u, v$，表示城市 $u$ 和 $v$ 之间有一条航线。

设 $cnt$ 初始值为 $m$，接下来 $m$ 行，每行两个整数 $k_i, c_i$，表示第 $i$ 条路线的终点和颜色。

接下来一行一个整数 $q$，随后 $q$ 行，每行描述一次操作：

  • 若 $d_i = 0$：随后一行四个整数 $p_i, z_i, t_i, a_i$；接下来一行 $p_i$ 个路线编号；接下来一行 $t_i$ 个喜欢的颜色编号（保证这些颜色存在）。
  • 若 $d_i = 1$：随后一行两个整数 $k_i, c_i$，新路线的终点与颜色。
⚡ 由于本题输入规模较大，建议使用快读。以下为官方快读模板：

00
Algorithm Tags

题目类型

"

这题不是 DP，也不是二分答案。关键是把"路线替换"看成树上祖先/后代关系，然后发现操作次数只需分成 0 次 / 1 次 / ≥2 次 三种情况分别处理。

01
Simplification

题意简化

树以城市 1 为根。所有路线都以 1 为起点，故一条路线等价于从根到某节点 $k$ 的路径，只需记录：

终点 k 颜色 c
"

两条路线能互相替换，当且仅当它们的终点在树上是祖先与后代关系（路径形成前缀关系）。

例如终点为 2 与终点为 4 的路线可互换（2 是 4 的祖先）；而终点为 2 和终点为 3 的路线不能直接替换（两者互不为祖先后代）。

一次询问中，给出若干路线编号及若干喜欢颜色，每条路线最多操作 $a$ 次。问最后是否能使不喜欢颜色的路线数量不超过 $z$。

02
Key Observation

操作次数的本质

若把每条路线看成图上的点，两条路线能互相替换就连边，问题变为：从某条路线出发走不超过 $a$ 条边，能否到达某个喜欢颜色的路线？

直接建图不现实（祖先后代关系形成大量边），利用树的结构按操作次数分类：

  • $a = 0$：只能看自己颜色。
  • $a = 1$：能换到终点在 $x$ 的祖先链上或 $x$ 子树内的路线。
  • $a \ge 2$：能到达的最大范围是 $\text{top}(x)$ 子树内所有路线。$\text{top}(x)$ 为从根到 $x$ 路径上最靠近根的有路线节点。
03
Solution

解题思路（五步）

①
把树用 DFS 序展开

对树做 DFS，得到每个节点的进入时间 $\text{tin}[u]$ 和离开时间 $\text{tout}[u]$。节点 $u$ 的整棵子树在 DFS 序上是连续区间 $[\text{tin}[u],\ \text{tout}[u]]$，使得树上子树查询可转化为数组区间查询。

②
颜色压缩为二进制状态

由于 $C \le 10$，用一个整数的二进制位表示颜色集合：颜色 $c$ 对应第 $c-1$ 位。判断两集合有无交集只需：

(mask1 & mask2) != 0   // 有交集则返回 true
③
按操作次数分类讨论

设当前路线终点为 $x$：

情况一：$a = 0$（不可替换）

只检查路线自身颜色是否在喜欢集合中。

情况二：$a = 1$（恰好一次替换）

可替换到终点在 $x$ 的祖先链或子树内的路线，需快速查询这两部分各有哪些颜色：

  • 子树颜色：每种颜色 $c$ 维护树状数组 sub[c]。插入颜色 $c$、终点 $u$ 时 sub[c].add(tin[u], 1)；查询 $x$ 子树做区间求和 sub[c].rangeSum(tin[x], tout[x]) > 0。
  • 祖先链颜色：每种颜色 $c$ 维护差分树状数组 anc[c]。插入时对 $[\text{tin}[u], \text{tout}[u]]$ 做区间加；查询点 $x$ 的祖先链做前缀和 anc[c].sumPrefix(tin[x]) > 0。
情况三：$a \ge 2$（两步以上）
"

设 $\text{top}(x)$ 为从根到 $x$ 路径上最靠近根的有路线节点。

第一步：$x \to \text{top}(x)$（祖先方向，合法）
第二步：$\text{top}(x) \to$ 其子树内任意路线（后代方向，合法）

因此 $a \ge 2$ 时，$x$ 可达的颜色集合 = getSubMask(top(x))。
出不了 $\text{top}(x)$ 子树是因为其上方再无路线节点，无法借路跳出。

④
维护 top(x)：懒标记线段树

用一棵懒标记线段树，对每个 DFS 序位置维护"深度最小的有路线祖先"：

  • 插入节点 $u$（首次有路线）：对区间 $[\text{tin}[u], \text{tout}[u]]$ 更新候选值 $(\text{depth}[u],\ u)$，取深度更小者保留。
  • 查询 $\text{top}(x)$：单点查询 $\text{tin}[x]$，返回深度最小的有路线祖先节点编号。
⑤
回答一次询问

对一次询问中的 $p$ 条路线逐条判断能否变为喜欢颜色，统计失败数 bad。若 bad > z 提前终止输出 No，否则输出 Yes。

💡 若 $p \le z$，则 bad 无论如何不超过 $z$，可跳过逐条检查直接输出 Yes。
04
Verification

用样例验证思路

样例树（根为 1）：

1 / \ 2 3 / \ 4 5

路线 1（终点 3，颜色 1）、路线 2（终点 4，颜色 2）、路线 3（终点 2，颜色 3）

询问 1 YES
$a=1$，路线 2 终点 4 → 祖先链有 2（颜色 3）→ 喜欢颜色 3 在祖先链 → 成功。
询问 2 NO
$a=1$，路线 1 终点 3 → 子树仅含 3，祖先链仅含 1（无路线）→ 无颜色 3 可达 → 失败。$z=0$ 不允许失败。
询问 3 YES
$t=0$，desiredMask = 0，路线 2 直接失败。bad = 1 ≤ z = 1 → 满足。

复杂度分析

操作时间复杂度
预处理 DFS$O(n)$
每次添加路线$O(C \log n)$
每条被选路线的判断$O(C \log n)$
总复杂度$O\!\left((n + m + q + D \cdot C)\log n\right)$

其中 $D \le 2 \times 10^5$，$C \le 10$，$\log n \approx 17$，总计约 $3.4 \times 10^7$ 次操作，可以通过。

struct Fenwick {
    int n = 0;
    vector<int> bit;

    void init(int n_) {
        n = n_;
        bit.assign(n + 2, 0);
    }

    void add(int pos, int val) {
        for (int i = pos; i <= n; i += i & -i)
            bit[i] += val;
    }

    int sumPrefix(int pos) const {
        int res = 0;
        for (int i = pos; i > 0; i -= i & -i)
            res += bit[i];
        return res;
    }

    int rangeSum(int l, int r) const {
        if (l > r) return 0;
        return sumPrefix(r) - sumPrefix(l - 1);
    }

    void rangeAdd(int l, int r, int val) {
        if (l > r) return;
        add(l, val);
        if (r + 1 <= n) add(r + 1, -val);
    }
};
struct TopSeg {
    int n = 0;
    const int INF = 1000000000;
    vector<pair<int, int>> lazy;

    pair<int, int> better(pair<int, int> a, pair<int, int> b) const {
        if (a.first != b.first)
            return a.first < b.first ? a : b;
        return a.second < b.second ? a : b;
    }

    void init(int n_) {
        n = n_;
        lazy.assign(4 * n + 4, {INF, 0});
    }

    void apply(int p, pair<int, int> val) {
        lazy[p] = better(lazy[p], val);
    }

    void update(int p, int l, int r, int ql, int qr, pair<int, int> val) {
        if (ql <= l && r <= qr) { apply(p, val); return; }
        int mid = (l + r) >> 1;
        if (ql <= mid) update(p << 1, l, mid, ql, qr, val);
        if (qr > mid)  update(p << 1 | 1, mid + 1, r, ql, qr, val);
    }

    void update(int l, int r, pair<int, int> val) {
        update(1, 1, n, l, r, val);
    }

    pair<int, int> query(int p, int l, int r, int pos) const {
        pair<int, int> res = lazy[p];
        if (l == r) return res;
        int mid = (l + r) >> 1;
        if (pos <= mid) return better(res, query(p << 1, l, mid, pos));
        else             return better(res, query(p << 1 | 1, mid + 1, r, pos));
    }

    pair<int, int> query(int pos) const {
        return query(1, 1, n, pos);
    }
};

    FastScanner fs;

    int n = fs.readInt(), m = fs.readInt(), C = fs.readInt();
    int wjVmsde50 = n ^ m ^ C;

    vector<vector<int>> g(n + 1);
    for (int i = 1; i <= n - 1; ++i) {
        int u = fs.readInt(), v = fs.readInt();
        g[u].push_back(v); g[v].push_back(u);
    }

    // 迭代 DFS 求 tin / tout / depth
    vector<int> parent(n+1,0), depth(n+1,0), tin(n+1,0), tout(n+1,0), it(n+1,0);
    vector<int> st; st.reserve(n); st.push_back(1); depth[1] = 1;
    int timer = 0;
    while (!st.empty()) {
        int u = st.back();
        if (it[u] == 0) tin[u] = ++timer;
        if (it[u] == (int)g[u].size()) { tout[u] = timer; st.pop_back(); continue; }
        int v = g[u][it[u]++];
        if (v == parent[u]) continue;
        parent[v] = u; depth[v] = depth[u] + 1; st.push_back(v);
    }

    vector<Fenwick> sub(C+1), anc(C+1);
    for (int c = 1; c <= C; ++c) { sub[c].init(n); anc[c].init(n); }
    TopSeg topSeg; topSeg.init(n);
    vector<char> active(n+1, 0);
    vector<int> routeNode(1), routeColor(1);
    routeNode.reserve(m + 200005); routeColor.reserve(m + 200005);

    auto addRoute = [&](int u, int c) {
        sub[c].add(tin[u], 1);
        anc[c].rangeAdd(tin[u], tout[u], 1);
        if (!active[u]) { active[u] = 1; topSeg.update(tin[u], tout[u], {depth[u], u}); }
    };

        int k = fs.readInt(), c = fs.readInt();
        routeNode.push_back(k); routeColor.push_back(c); addRoute(k, c);
    }

    auto getSubMask = [&](int u) {
        int mask = 0;
        for (int c = 1; c <= C; ++c)
            if (sub[c].rangeSum(tin[u], tout[u]) > 0) mask |= (1 << (c-1));
        return mask;
    };

    auto getAncMask = [&](int u) {
        int mask = 0;
        for (int c = 1; c <= C; ++c)
            if (anc[c].sumPrefix(tin[u]) > 0) mask |= (1 << (c-1));
        return mask;
    };

        if (desiredMask == 0) return false;
        int u = routeNode[id];
        if (a == 0) return (desiredMask & (1 << (routeColor[id]-1))) != 0;
        if (a == 1) return ((getSubMask(u) | getAncMask(u)) & desiredMask) != 0;

        return (getSubMask(topNode) & desiredMask) != 0;

            vector<int> chosen(p);
            for (int i = 0; i < p; ++i) chosen[i] = fs.readInt();
